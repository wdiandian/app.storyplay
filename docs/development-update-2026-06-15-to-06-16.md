# 2026-06-15 至 2026-06-16 开发更新

## 概览

这两天的重点，不是继续堆字段，而是把项目从“能跑的互动影游 Demo”推进到“更像平台雏形的互动影游编辑器 + 更像正式产品的玩家端”。

本轮完成的核心方向：

- 后台从配置页升级为可工作的影游编辑器
- 前台从普通视频页升级为更强沉浸感的互动影游体验
- 运行时规则、片段试玩、片段删除、共享存储链路进一步补齐

## 后台编辑器

### 1. 编辑器结构重构

后台不再围绕某个预设作品字段展开，而是拆成更通用的工作区：

- 项目配置
- 素材与发布
- 搭建剧情
- 编辑片段
- 设置出口

同时补了：

- 左侧工作流导航
- 左侧片段导航
- 片段搜索与筛选
- 列表视图 / 章节视图
- 问题片段分组
- 项目进度检查与风险摘要

### 2. 片段工作流增强

围绕“片段”补齐了更像编辑器的动作：

- 创建片段
- 复制片段
- 删除片段
- 设为起始片段
- 试玩当前片段
- 进入片段编辑
- 进入出口编辑

其中删除片段不是简单删除，会自动清理引用关系：

- 片段出口目标
- 自动跳转目标
- 时间线 `jump` 目标
- 时间线 `show_choice` 中的视频内选项目标

如果删除的是起始片段，会自动回退到剩余第一个片段。
如果只剩最后一个片段，则禁止删除，避免项目结构失效。

### 3. 时间线编辑器增强

时间线从普通事件表单，推进到更像可操作编辑器的状态：

- 轨道预览
- 事件摘要卡片
- 条件 / 动作数量摘要
- 事件复制
- 时间微调
- 按触发时间排序
- 拖拽重排事件

### 4. 术语与信息架构继续通用化

继续清理过于定制化或容易出戏的后台表达，重点包括：

- 逐步把“场景”统一为“片段”
- 把“目标场景”统一为“目标片段”
- 弱化预设作品痕迹，改成更平台化的字段与文案

## 前台玩家端

### 1. 入口页重做

前台首页已经不是简单视频页，而是加入了更正式的入口结构：

- 宣传 / 预告入口页
- 作品标题与导语
- 入口视频与封面
- 进入正片前的作品说明

### 2. 播放体验升级

播放器改成更偏“舞台感 / 游戏感”的呈现：

- 选项直接压在视频画面上
- 关键时刻突出决策感
- 倍速播放
- 信息面板可展开 / 收起
- 右侧面板可关闭
- 画面结束后停留最后一帧，不再黑屏

### 3. 片段试玩模式

前台支持从指定片段直接开始试玩：

- `/?previewNode=片段编码`

后台“试玩当前片段”已经直接接到这条链路。
在片段试玩模式下重开时，也会继续从该片段开始。

### 4. 结局与状态展示优化

继续收口玩家可见信息，减少出戏字段，并强化：

- 结局呈现
- 当前片段状态
- 选择反馈
- 作品说明区

## 数据与运行时

### 1. 共享存储链路继续补齐

项目现在已经支持：

- SQLite 本地存储
- PostgreSQL 远端共享存储

并通过统一存储层做切换，减少业务层分叉。

### 2. 规则系统与运行时联动

规则系统已经从数据结构、后台编辑到前台运行时贯通，包括：

- 变量定义
- 条件
- 动作
- 时间线事件
- 时间线内动态选项

### 3. Playthrough 能力增强

补齐了试玩 / 重开时的运行时能力：

- 从项目起始片段创建试玩
- 从指定片段创建试玩
- 从指定片段重开
