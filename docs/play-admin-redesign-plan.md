# 播放页与创作后台改造方案

本文档分析当前播放页和创作后台的问题，并提出下一阶段改造方案。当前结论：先做播放页体验壳改造，再进入创作后台画布化改造。

## 1. 优先级判断

建议优先顺序：

1. 播放页。
2. 创作后台画布。

原因：

- 首页已经改成发现入口，用户下一步自然会进入播放页；如果播放页仍像普通视频页，产品主体验会断掉。
- 播放页第一轮改造主要是界面结构和交互呈现，不需要立刻改数据模型，风险较低。
- 创作后台画布化会影响编辑模型、组件结构、后续 API 和保存逻辑，是更大的改造，需要先定清楚画布交互和数据边界。

但后台方向要现在同步确定，否则播放页后续需要的“互动感”会和后台可编辑能力脱节。

## 2. 播放页问题分析

当前播放页已经有暗色、转场、选项卡和结局面板，但整体仍然像“视频播放页”，主要问题不是样式不够，而是体验模型不够互动影游。

### 2.1 预告页割裂

当前进入作品后先看到一个预告片卡片和右侧说明卡片。这个结构更像内容详情页，不像游戏开场。

问题：

- 玩家还没进入故事，就先进入了一个双栏信息页。
- 视频被包在卡片里，沉浸感被削弱。
- “开始进入正片”“播放 / 暂停视频”“倍速”这些按钮让页面更像播放器工具。

目标：

- 首屏直接进入电影化开场。
- 宣传信息作为开场叠层出现，而不是独立详情页。
- 开始按钮应该像“进入故事”，不是“播放视频”。

### 2.2 原生视频控件太强

当前正片视频仍使用浏览器原生 controls。

问题：

- 原生控件会让玩家感知变成普通视频。
- 倍速、跳过、重开、信息面板同时出现，工具感太强。
- 玩家注意力从剧情转移到播放器控制。

目标：

- 默认隐藏原生 controls。
- 使用自定义极简控制：播放/暂停、静音、重开。
- 倍速、跳过、信息面板降级为二级操作。

### 2.3 互动发生得太晚

当前主要交互是视频结束后出现选项，或者时间线事件触发叠层。

问题：

- 玩家大部分时间只是在看视频。
- 选择像“下一集按钮”，不像剧情里的决策。
- 时间线事件虽然存在，但玩家感知不够强。

目标：

- 把互动拆成“观看、提示、抉择、反馈、转场、后果”。
- 选择出现时冻结或压暗画面，让它成为剧情事件。
- 选择后立即给反馈：已选择、影响、进入下一幕。

### 2.4 信息面板偏调试

当前信息面板包含选择记录、作品信息、提示记录，功能是有价值的，但表达偏工具。

问题：

- “内容面板”“state/actions”这类结构更像调试面板。
- 玩家不需要理解运行时状态，只需要感知自己的路径和后果。

目标：

- 信息面板改成“回忆 / 线索 / 状态”这类剧情化面板。
- 变量和动作不要直接暴露为技术状态。
- 只展示对玩家有意义的选择、线索和结局摘要。

### 2.5 节点之间缺少连续性

当前每个节点是一个视频片段，节点切换虽然有转场，但玩家感知仍像“播放下一个视频”。

目标：

- 转场文案表达剧情推进，而不是只显示第几幕。
- 选择后在转场中显示选择结果。
- 进入下一节点时保留上一选择造成的情绪或提示。

## 3. 播放页目标方案

播放页目标不是“更漂亮的视频播放器”，而是“互动影游运行时”。

### 3.1 页面结构

目标结构：

1. 全屏视频舞台。
2. 顶部极简状态条。
3. 底部剧情标题和当前提示。
4. 分支出现时的决策层。
5. 右侧或底部的剧情化信息抽屉。
6. 结局结算层。

去掉：

- 独立预告详情页。
- 大量工具按钮常驻。
- 原生视频控件常驻。

### 3.2 互动节奏

标准体验节奏：

1. 开场叠层：作品名、短导语、开始。
2. 视频播放：界面尽量隐藏。
3. 时间线事件：短提示或剧情叠层。
4. 分支抉择：画面冻结或压暗，选择成为主视觉。
5. 选择反馈：显示玩家刚刚做出的选择。
6. 转场：进入下一幕。
7. 结局：展示结局、路径摘要、重新开始。

### 3.3 第一阶段改造范围

第一阶段只做体验壳，不改 API 和数据模型。

改造内容：

- 正片改成全屏舞台。
- 去掉原生 controls，改成极简自定义控制。
- 预告页合并为开场叠层。
- 选择层改得更像“剧情决策”，减少卡片工具感。
- 信息面板改名和改文案，降低技术感。
- 选择后转场展示本次选择。

暂不做：

- 新时间线事件类型。
- 复杂变量可视化。
- 发布版本绑定。
- 多结局成就系统。

## 4. 创作后台问题分析

当前后台的问题不是功能少，而是功能太多都以表单方式出现。创作者需要理解项目设置、节点、选择、变量、条件、动作、时间线、发布检查，这对互动影游创作不友好。

### 4.1 表单优先导致认知负担高

当前后台的主要工作方式是：

- 先选项目。
- 再选工作区。
- 再选节点。
- 再进入大量字段。
- 再编辑选择、时间线、变量和条件。

问题：

- 创作者看不到“故事结构”是主对象。
- 很多字段需要先理解技术含义。
- 新手会不知道下一步该做什么。

目标：

- 画布是主入口。
- 节点和连线是主对象。
- 表单只是选中对象后的属性面板。

### 4.2 分支图被放成辅助工具

此前已有旧分支图组件，但它只是剧情工作台里的一块内容。

问题：

- 图不是主工作区。
- 图上能点节点、加后续、加分支，但它没有成为完整编辑入口。
- 真实编辑仍然要回到下方复杂表单。

目标：

- 画布居中，占据最大区域。
- 节点卡片直接显示状态、视频、出口、问题。
- 拖拽或点击创建连线。
- 选中节点后右侧检查器编辑内容。

### 4.3 编辑对象层级混乱

当前项目、节点、选择、时间线、变量、条件、动作都在一个大组件中处理。

问题：

- 组件复杂，难维护。
- 用户也容易混淆“玩家可见内容”和“逻辑配置”。
- 时间线 payload 仍有 JSON 编辑痕迹。

目标：

- 项目设置放全局面板。
- 节点内容放节点检查器。
- 选择和自动跳转放连线检查器。
- 时间线事件放节点内部时间轴。
- 变量、条件、动作放高级抽屉。

## 5. 创作后台目标方案

后台目标是“画布式互动影游编辑器”。

### 5.1 页面结构

目标结构：

1. 顶部工具栏：项目切换、保存、预览、发布检查、导入导出。
2. 左侧资源栏：项目结构、素材、变量、问题列表。
3. 中央画布：节点、连线、分支、状态。
4. 右侧检查器：当前选中对象的编辑面板。
5. 底部可选时间轴：当前节点的视频时间线事件。

### 5.2 画布对象

画布上的主对象：

- 视频节点。
- 结局节点。
- 自动跳转线。
- 玩家选择线。
- 问题标记。

节点卡片显示：

- 标题。
- 缩略图或视频状态。
- 节点类型。
- 入度/出口数量。
- 是否起点。
- 是否缺视频。
- 是否缺出口。

连线显示：

- 自动跳转。
- 玩家选项。
- 缺失目标。
- 条件分支。

### 5.3 检查器

右侧检查器根据选中对象切换：

- 选中项目：展示信息、首页展示、宣传视频、全局变量。
- 选中节点：标题、描述、视频、正文、节点类型、结局设置。
- 选中连线：选择文案、提示、目标节点、条件、动作。
- 选中时间线事件：触发时间、类型、文案、目标、条件、动作。
- 未选中：展示发布检查、最近问题、快捷创建。

### 5.4 技术路线

建议分两步：

第一步曾考虑用旧分支图做“画布主工作区”，当前已改为独立画布 V2 编辑器。

- 风险低。
- 不新增依赖。
- 可以先重排布局，把图提升为主入口。
- 保留现有保存 API。

第二步：如果画布交互要继续增强，考虑引入 React Flow。

- 更适合拖拽节点、连线编辑、缩放、平移、选择状态。
- 更符合长期的画布编辑器方向。
- 需要新增依赖和重新设计图数据适配层。

## 6. 建议实施顺序

### 阶段 A：播放页体验壳

目标：让玩家进入作品后第一感觉就是互动影游。

任务：

- 合并预告页到开场叠层。
- 去掉正片原生 controls。
- 增加极简控制层。
- 重做选择层。
- 选择后展示反馈转场。
- 信息面板改成剧情化抽屉。

验收：

- 打开播放页不像普通视频详情页。
- 播放中 UI 干扰明显减少。
- 出现选择时玩家能明确感到剧情暂停并等待决策。
- 选择后有明确反馈。

### 阶段 B：后台画布入口

目标：让画布成为后台主工作区。

任务：

- 后台默认进入剧情画布。
- 左侧从复杂导航收敛成项目/素材/问题。
- 中央放大旧分支图。
- 右侧做节点检查器雏形。
- 项目设置降级为全局面板。

验收：

- 创作者第一眼看到故事结构。
- 点击节点即可编辑节点。
- 新建节点和创建分支从画布出发。
- 不需要先进入复杂表单才能理解项目。

详细设计见 [创作后台画布设计](./creator-canvas-design.md)。

### 阶段 C：后台画布增强

目标：把图从“可视化图谱”推进到“真正可编辑画布”。

任务：

- 节点拖拽和布局保存。
- 连线创建和连线检查器。
- 时间线事件可视化。
- 条件和动作结构化控件。
- 发布检查问题定位到画布对象。

验收：

- 创作者能主要通过画布搭建剧情。
- 表单只作为属性编辑器存在。
- JSON 编辑只保留为高级模式或调试模式。

## 7. 当前不做

- 不立即重写数据模型。
- 不立即引入账号系统。
- 不立即做多人协作。
- 不立即做复杂素材库。
- 不在播放页第一轮改造中新增复杂玩法系统。
