# StoryPlay编辑器下一阶段方案

## 目标

把当前后台从“可编辑数据的管理页”继续升级为“更完整的影游编辑器”。

这一阶段的重点不是继续堆字段，而是让创作者能按更自然的创作心智工作：

1. 先看项目整体
2. 再搭剧情结构
3. 再编辑单个片段
4. 再编排互动
5. 最后统一检查素材与发布风险

## 当前已经落地的骨架

目前后台已经具备以下基础：

- 左侧固定导航
  - 工作流阶段入口
  - 片段列表
  - 快速操作
- 右侧主工作区
  - 当前工作区摘要
  - 项目进度检查
  - 各阶段编辑面板
- 片段工作流拆分
  - 项目配置
  - 搭建剧情
  - 编辑片段
  - 设置出口
  - 素材与发布

这套骨架已经比传统表单后台更接近编辑器，但还不是完整的影游创作台。

## 下一阶段建议拆分

### 阶段 A：片段编辑器深化

目标：让“编辑片段”页真正成为高频工作区。

建议增加：

- 片段头部工具条
  - 保存
  - 预览当前片段
  - 复制片段
  - 删除片段
- 片段信息区分组
  - 基础信息
  - 播放内容
  - 结局设置
  - 自动跳转
- 片段状态提示
  - 缺视频
  - 无出口
  - 未设标题
  - 时间线未保存

### 阶段 B：时间线编辑器升级

目标：让时间线从“事件表单”升级为更像 NLE / 互动时间轴。

建议增加：

- 时间轴标尺
  - 秒级刻度
  - 当前总时长
  - 事件分布可视化
- 事件轨道分层
  - 文本轨
  - 选项轨
  - 叠层轨
  - 跳转轨
  - 动作轨
- 事件卡片摘要
  - 触发时间
  - 类型
  - 标题/目标
  - 条件数量
  - 动作数量
- 事件复制能力
  - 复制当前事件
  - 批量排序

### 阶段 C：分支与结构视图升级

目标：让“搭建剧情”不只是图，而是可操作的结构面板。

建议增加：

- 结构视图切换
  - 剧情图
  - 列表视图
  - 风险视图
- 风险筛选
  - 仅看缺出口
  - 仅看无效目标
  - 仅看未设视频
  - 仅看孤立片段
- 快速批处理
  - 从列表跳转到片段
  - 从风险项直接修复

### 阶段 D：素材中心

目标：把素材上传从“填 URL”升级为真正的素材面板。

建议增加：

- 素材分类
  - 入口视频
  - 入口封面
  - 片段视频
  - 角色图
  - 宣传图
- 素材引用关系
  - 哪个片段在用
  - 哪个入口页在用
- 素材预览卡
  - 缩略图
  - 文件名
  - 文件大小
  - 最近上传时间

### 阶段 E：面向平台化的配置层

目标：支持“不是单个游戏定制后台”，而是通用StoryPlay平台。

建议增加：

- 可配置展示字段
  - 标题
  - 简介
  - 宣传说明
  - 片段说明
- 通用变量系统
  - 标签型
  - 数值型
  - 布尔型
  - 枚举型
- 内容模块扩展
  - 角色卡
  - 世界观卡
  - 宣传图集
  - 章节封面

## 推荐实现顺序

建议按下面顺序推进：

1. 先做片段编辑器深化
2. 再做时间线编辑器升级
3. 再做分支与结构视图升级
4. 然后做素材中心
5. 最后做平台化配置层

原因：

- 片段编辑和时间线是最高频操作
- 结构图和素材中心属于中频操作
- 平台化配置层要建立在稳定编辑器骨架之上

## 本轮之后的直接可做项

如果继续往下开发，最值得立即落地的是这 4 项：

1. 左侧片段列表支持搜索、筛选、起始片段标识
2. 编辑片段页增加顶部工具条和未保存提示
3. 时间线事件增加更强的卡片摘要和复制能力
4. 搭建剧情页增加“风险片段列表”

## 设计原则

后续所有后台设计建议遵守这几条：

- 少讲概念，多给操作入口
- 少暴露技术字段，多暴露创作结果
- 高频操作集中到一个工作区
- 低频配置折叠到次级面板
- 玩家看不到的字段，尽量不打断创作者主流程
