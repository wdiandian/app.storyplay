# 产品概览

## 定位

StoryPlay App 是一个附属的互动影游创作与播放项目。

主站负责 AI 实时生成互动影游；本项目负责手工制作的分支影游、经典/模板作品展示，以及创作者编辑工具。

## 核心闭环

1. 创作者在创作后台新建或编辑游戏。
2. 创作者配置项目资料、节点、选择、变量、时间线事件和媒体资源。
3. 合格的游戏展示到首页。
4. 玩家在首页发现游戏，进入播放页。
5. 玩家观看视频节点并做出选择，直到抵达结局。

## 用户角色

创作者：

- 管理项目；
- 编辑项目信息和媒体；
- 搭建节点、分支和条件；
- 在公开展示前测试项目。

玩家：

- 浏览已展示游戏；
- 开始一次游玩；
- 做出分支选择；
- 抵达并回顾结局。

运营者：

- 部署和监控应用；
- 管理存储和备份；
- 后续负责发布策略、可见性、审核和统计。

## 产品板块

首页：

- 展示已上架或已设置为首页展示的游戏；
- 只依赖项目摘要数据；
- 不暴露草稿态、编辑器内部状态和复杂管理能力。

播放页：

- 承载单个游戏的玩家运行时；
- 处理视频播放、分支选择、自动跳转、时间线事件、条件、动作和结局；
- 对缺失节点、缺失视频、空项目等异常提供清晰状态。

创作后台：

- 管理项目和编辑游戏；
- 重视结构正确性、编辑效率和发布前检查；
- 提供项目总览、节点编辑、分支编辑、变量/条件/动作、时间线事件、导入导出等能力。

基建：

- 领域模型；
- 存储适配；
- API 契约；
- 上传和资源访问；
- 部署脚本；
- 后续账号、发布流、权限、统计和审核。

## 非目标

- 不承担主站的 AI 实时生成入口。
- 不做营销型落地页。
- 不与主站仓库合并。
- 不在当前创作/播放闭环稳定前重写整个运行时。
