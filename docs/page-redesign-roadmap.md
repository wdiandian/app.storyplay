# 页面改造 Roadmap

本文档描述 StoryPlay App 页面改造的阶段计划。当前优先改造首页；播放页和创作后台先保留方向，不在本阶段展开具体设计。

## 1. 总目标

把项目从“功能已经能跑的原型”推进到“结构清晰、视觉稳定、后续可持续扩展的互动影游产品”。

页面分为三个主板块：

- 首页：玩家发现游戏。
- 播放页：玩家游玩游戏。
- 创作后台：创作者编辑游戏。

本轮页面改造遵循：

- 先改页面结构，再补视觉细节。
- 先保证可用和稳定，再追求完整品牌感。
- 不在 UI 改造中混入数据模型大迁移。
- 每个阶段都能单独构建和部署。

## 2. 阶段 1：首页改造

目标：

- 首页从“宣传片流”调整为“影游发现页”。
- 让玩家第一屏看到一个精选作品。
- 让玩家向下可以浏览更多作品。
- 保留创作后台入口，但不抢玩家入口。
- 不改动现有数据接口。

使用数据：

- `ProjectSummary.title`
- `ProjectSummary.tagline`
- `ProjectSummary.slug`
- `ProjectSummary.promoVideoUrl`
- `ProjectSummary.promoPosterUrl`
- `ProjectSummary.updatedAt`
- `ProjectSummary.sortOrder`
- `ProjectSummary.listedOnHome`

页面结构：

1. 顶部导航。
2. 精选作品首屏。
3. 作品列表。
4. 空状态。

验收标准：

- 首页只展示 `listedOnHome` 项目。
- 有项目时，首个排序最高的项目进入精选区。
- 其他项目以卡片列表展示。
- 无项目时，有明确空状态和进入后台路径。
- 桌面端和移动端文字不重叠。
- `npm run build` 通过。

## 3. 阶段 2：播放页方向

播放页后续再单独讨论具体设计。本阶段只保留方向：

- 暗色沉浸式视频运行时。
- 视频内容优先。
- 分支选择明确、低干扰。
- 结局、错误、重试状态完整。
- 不把后台编辑信息暴露给玩家。

暂不做：

- 播放状态机重写。
- 时间线事件系统重构。
- 发布版本绑定。

## 4. 阶段 3：创作后台方向

创作后台后续再单独讨论具体设计。本阶段只保留方向：

- 明亮、中性、高密度工作台。
- 左侧目录、中间编辑、右侧上下文。
- 保存、预览、发布检查清晰。
- 节点、选择、变量、时间线事件逐步结构化。

暂不做：

- 继续维护旧表单编辑器。
- 新增账号权限。
- 破坏性数据库迁移。

## 5. 首页之后的建议顺序

1. 对首页做一次浏览器视觉检查。
2. 补首页空状态和项目卡片细节。
3. 先做播放页体验壳改造，让玩家入口形成完整闭环。
4. 再做创作后台画布入口，让编辑体验从表单工作流转向画布工作流。
5. 最后进入后台画布增强、结构化时间线和发布检查。

首页改造完成后，不代表整体视觉最终完成；它只是先建立玩家入口的产品形态。

播放页和创作后台的详细分析见 [播放页与创作后台改造方案](./play-admin-redesign-plan.md)。
